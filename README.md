# tweet-watcher 🐦🔔

מנטר את תוצאות החיפוש ב-[twitterwebviewer.com](https://twitterwebviewer.com) לפי מילות מפתח שתגדיר,
רץ **כל 10 דקות** דרך GitHub Actions (חינם), ושולח **התראה לבוט טלגרם** על כל ציוץ חדש.

## איך זה עובד

```
GitHub Actions (cron כל 10 דק')
        │
        ▼
  פותח דפדפן (Playwright) → נכנס ל-twitterwebviewer.com לכל מילת מפתח (מיון: Latest)
        │
        ▼
  משווה מזהי ציוצים מול state/seen.json  ← "מה כבר ראיתי"
        │
        ▼
  כל ציוץ חדש → שולח הודעה לבוט הטלגרם שלך
        │
        ▼
  שומר את המצב המעודכן בחזרה ל-state/seen.json
```

**מילות המפתח פרטיות** — הן נשמרות ב-Secret בשם `KEYWORDS` (מופרדות בפסיק או בשורה חדשה),
כדי שהרשימה לא תיחשף גם ב-repo ציבורי. קובץ ה-state מגובב (hash) והלוגים מדפיסים רק את הגיבוב,
כך שהמילה עצמה לעולם לא נכתבת ל-repo או ללוגים — היא נשלחת רק לטלגרם הפרטי שלך.

כדי לשנות מילות מפתח: **Settings → Secrets and variables → Actions → `KEYWORDS`** → Update.
(לפיתוח מקומי אפשר במקום זה קובץ `keywords.json` — הוא מוחרג מ-git ולא נשמר ב-repo.)

## שלב 1 — יצירת בוט טלגרם והשגת הפרטים

1. פתח בטלגרם צ'אט עם **[@BotFather](https://t.me/BotFather)**.
2. שלח `/newbot`, בחר שם ו-username לבוט. בסוף תקבל **Bot Token** שנראה כך:
   `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` — שמור אותו.
3. עכשיו צריך את ה-**Chat ID** (לאן לשלוח את ההתראות — אליך):
   - שלח הודעה כלשהי (למשל `שלום`) **לבוט שיצרת** (חפש אותו לפי ה-username ולחץ Start).
   - פתח בדפדפן:
     `https://api.telegram.org/bot<TOKEN>/getUpdates`
     (החלף את `<TOKEN>` בטוקן שלך).
   - חפש בתגובה שדה `"chat":{"id":123456789,...}` — המספר הזה הוא ה-**Chat ID** שלך.
   - אם מקבלים `{"ok":true,"result":[]}` ריק — שלח קודם הודעה לבוט ואז רענן.

> רוצה שההתראות יגיעו לקבוצה/ערוץ במקום אליך אישית? הוסף את הבוט לקבוצה, שלח שם הודעה, ואז ה-`getUpdates` יראה את ה-`chat id` של הקבוצה (מתחיל במינוס, למשל `-1001234...`).

## שלב 2 — העלאה ל-GitHub

1. צור repo חדש ב-GitHub (למשל `tweet-watcher`).
2. העלה אליו את כל הקבצים מהתיקייה הזו. אם אין לך Git מותקן, אפשר פשוט לגרור את הקבצים
   בממשק הווב של GitHub (Add file → Upload files). ודא שגם התיקיות `.github/` ו-`src/` עולות.
3. הוסף את הסודות (Secrets):
   **Settings → Secrets and variables → Actions → New repository secret**, צור שניים:
   - `TELEGRAM_BOT_TOKEN` — הטוקן מ-BotFather
   - `TELEGRAM_CHAT_ID` — ה-Chat ID שמצאת
4. הפעל Actions: לשונית **Actions** ב-repo → אשר את ההרצה אם GitHub מבקש.
5. בדיקה מיידית: **Actions → tweet-watcher → Run workflow** (הרצה ידנית).
   בהרצה הראשונה תקבל בטלגרם הודעת "מתחיל לנטר…" לכל מילת מפתח, ומכאן והלאה — רק ציוצים חדשים.

## ⚠️ שים לב (חשוב)

- **מכסת דקות (repo פרטי):** GitHub נותן 2000 דקות Actions חינם בחודש ל-repo פרטי.
  הרצה כל 10 דק' עלולה לחרוג מזה. שתי אפשרויות:
  - **repo ציבורי (public)** → דקות Actions **ללא הגבלה, חינם**. הטוקן נשמר ב-Secrets ולא נחשף.
    (מילות המפתח והציוצים שנשמרים ב-`state` יהיו גלויים — לרוב לא בעיה כי אלה ציוצים ציבוריים.)
  - להשאיר את ה-repo פרטי אבל להעלות את התדירות ל-`*/30` (כל חצי שעה) כדי להישאר בתוך המכסה.
    (ערוך את השורה `cron` ב-[`.github/workflows/watch.yml`](.github/workflows/watch.yml).)
- **התזמון הוא "מאמץ מיטבי":** GitHub לפעמים מעכב הרצות מתוזמנות בכמה דקות. זה לא שעון מדויק,
  אבל בפועל תקבל את הציוצים בפער של דקות בודדות.
- **תלות באתר צד-שלישי:** twitterwebviewer.com הוא מתווך לא-רשמי. אם הם ישנו את מבנה הדף
  או ייחסמו מול טוויטר — הסורק עלול להפסיק לעבוד עד עדכון. אם זה קורה, שקול API בתשלום
  (twitterapi.io / Apify) שיציב יותר.
- **הצפה בהרצה בודדת:** יש מנגנון שמגביל ל-10 התראות למילת מפתח בכל הרצה, כדי לא להישלח מבול.

## הרצה מקומית (לבדיקה על המחשב שלך)

דרוש Node.js 20+.

```bash
npm install
npx playwright install chromium

# Windows PowerShell:
$env:TELEGRAM_BOT_TOKEN="<token>"; $env:TELEGRAM_CHAT_ID="<chatid>"; node src/scrape.mjs
```

בלי משתני הסביבה — הסקריפט ידפיס את ההתראות למסך במקום לשלוח לטלגרם (נוח לבדיקה).

---

## מבנה הפרויקט

| קובץ | תפקיד |
|------|--------|
| `KEYWORDS` (Secret) | מילות המפתח לניטור — פרטיות |
| `src/scrape.mjs` | הסורק + שליחה לטלגרם |
| `state/seen.json` | זיכרון של ציוצים שכבר נשלחו (ממופתח לפי hash, מתעדכן אוטומטית) |
| `.github/workflows/watch.yml` | התזמון של GitHub Actions (כל 10 דק') |
| `keywords.example.json` | דוגמה לפורמט לפיתוח מקומי בלבד |
